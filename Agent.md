# Agent Context - Aura AI Landing Page

## Overview

**Project**: Aura AI - Professional landing page for an intelligent automation agency targeting Puerto Rico businesses.

**Core Value Proposition**: "We don't sell software, we deliver transformation." The site presents AI automation services with a focus on local Puerto Rico market, bilingual support (Spanish/English), and measurable results for clinics, law firms, retailers, and other local businesses.

**Business Context**: Based on strategic market research (`investigacionmercado.md`), the agency positions itself as a hybrid high-value service (Done-With-You/Done-For-You) with premium pricing ($1,500-$5,000+ setup, $297-$497/month). Key differentiators: local Puerto Rico presence, bilingual support, training, and custom development.

## Coding Patterns & Conventions

### Component Architecture
1. **Functional Components** with TypeScript interfaces for props
2. **Default exports** for components
3. **Named imports** from `@/` alias
4. **Organization**: Layout, sections, and UI primitives separated

### TypeScript Usage
- Explicit interfaces for component props
- Type-safe routes with TanStack Router
- Strict TypeScript configuration (no implicit any)

### Styling Patterns
1. **Tailwind CSS** for utility-first styling
2. **Custom theme variables** for consistent colors, spacing, typography
3. **Glassmorphism** effects via custom utilities (`glass`, `glass-button`)
4. **Responsive design** with custom breakpoints
5. **Dark mode only** (color-scheme: dark)

### Internationalization Pattern
```tsx
import { useTranslation } from 'react-i18next'

export default function Component() {
  const { t } = useTranslation()
  return <div>{t('namespace.key')}</div>
}
```

**Translation keys** follow component/section naming (e.g., `header.brand`, `hero.titleLine1`).

### Animation Patterns
```tsx
import { motion } from 'framer-motion'
import { fadeInUp } from '@/lib/animations'

<motion.div {...fadeInUp}>
  {/* content */}
</motion.div>
```

### Image & Asset Handling
- **Static images**: Place in `public/images/` and reference directly
- **Icons**: Use Lucide React icons (`import { Menu } from 'lucide-react'`)
- **Favicons**: Multiple variants in `public/favicon/`

### State Management
- **Local state**: `useState`, `useEffect`
- **Server state**: React Query (configured but minimal usage currently)
- **No global state library** (context/redux) - not needed for static landing page

## Agent Guidelines

### When Working on This Project

#### DO:
1. **Follow existing patterns**: Component structure, styling approach, i18n usage
2. **Use TypeScript strictly**: No `any` types, define proper interfaces
3. **Maintain bilingual support**: All new text must be added to both `en/translation.json` and `es/translation.json`
4. **Preserve design consistency**: Use theme variables, glass effects, animations
5. **Test responsive behavior**: Check mobile (809px), tablet (810px), desktop (1200px) breakpoints
6. **Run linting**: `npm run lint` before considering work complete
7. **Verify builds**: `npm run build` should succeed without errors

#### DON'T:
1. **Introduce new dependencies** without discussion
2. **Change design tokens** (colors, fonts, breakpoints) arbitrarily
3. **Break existing i18n structure** - follow nested key patterns
4. **Use inline styles** when Tailwind utilities exist
5. **Commit without linting** and type checking

### Common Tasks & Approaches

#### Adding a New Component
1. Determine category (layout, section, ui)
2. Create file in appropriate directory
3. Define TypeScript interface for props
4. Use existing styling patterns (Tailwind + theme variables)
5. Add animations via Framer Motion presets
6. Implement i18n with translation keys
7. Export as default

#### Modifying Content
1. Locate translation keys in JSON files
2. Update both English and Spanish versions
3. Maintain consistent tone (professional, benefit-focused, local relevance)
4. Preserve formatting and structure

#### Adding a New Page/Section
1. Consider if it fits existing section component pattern
2. Create new component in `src/components/sections/`
3. Add to appropriate route (homepage) or create new route
4. Update translations with new namespace
5. Ensure responsive design

#### Styling Changes
1. Use Tailwind utilities first
2. For reusable patterns, add to `@layer utilities` in `src/index.css`
3. For theme extensions, add to `@theme` block
4. Test across breakpoints

### Testing & Verification
1. **Development server**: `npm run dev` - verify changes visually
2. **Type checking**: Part of `npm run build`
3. **Linting**: `npm run lint`
4. **Production build**: `npm run build` should complete without errors
5. **Preview**: `npm run preview` to verify production build

### Performance Considerations
- Images optimized via Vite
- Code splitting via TanStack Router
- Vendor chunking for React/Framer Motion
- Minimal JavaScript bundle for landing page
